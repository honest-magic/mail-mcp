# Phase 27 Plan 01: Delete Email Tool

## Objective

Add a `delete_email` MCP tool that permanently deletes a single email by UID
from a specified folder, with body-cache invalidation and read-only mode
enforcement.

## Context

@.planning/phases/27-delete-email-tool/27-CONTEXT.md

## Tasks

### Task 1 — Add `deleteMessage` to ImapClient (TDD)
type: auto
tdd: true

**Behavior:**
- `ImapClient.deleteMessage(uid: string, folder: string): Promise<void>`
- Calls `this.batchDeleteMessages([uid], folder)` internally
- Throws `Error('Not connected')` when client is null (consistent with other methods)

**Test file:** `src/protocol/imap.delete.test.ts`

**Tests:**
1. `deleteMessage` calls `batchDeleteMessages` with single-element array
2. `deleteMessage` passes correct folder argument
3. `deleteMessage` throws when not connected

**Implementation file:** `src/protocol/imap.ts`
Add `deleteMessage(uid: string, folder: string): Promise<void>` after `batchMoveMessages`.

### Task 2 — Add `deleteEmail` to MailService (TDD)
type: auto
tdd: true

**Behavior:**
- `MailService.deleteEmail(uid: string, folder: string = 'INBOX'): Promise<void>`
- Calls `this.imap.deleteMessage(uid, folder)`

**Test file:** `src/services/mail.delete.test.ts`

**Tests:**

2. `deleteEmail` invalidates the body cache after deletion
3. `deleteEmail` defaults folder to 'INBOX' when not provided

**Implementation file:** `src/services/mail.ts`

### Task 3 — Wire `delete_email` tool in src/index.ts
type: auto

**Changes:**
1. Add `'delete_email'` to `WRITE_TOOLS` set
2. Add tool definition in `getTools()` after `batch_operations`
3. Add handler branch in `dispatchTool()` (new-style) and in the
   `CallToolRequestSchema` handler (legacy-style)
4. Update `instructions` string in `MailMCPServer` constructor to include
   `delete_email` in the read-only mode description

**Tool definition:**
```json
{
  "name": "delete_email",
  "description": "Permanently delete a single email by UID via IMAP — works with any provider (Gmail, Outlook, custom domains). This is irreversible; prefer move_email to Trash if you want recovery.",
  "annotations": { "readOnlyHint": false, "destructiveHint": true },
  "inputSchema": {
    "type": "object",
    "properties": {
      "accountId": { "type": "string", "description": "The ID of the account to use" },
      "uid": { "type": "string", "description": "The UID of the email to delete" },
      "folder": { "type": "string", "description": "The folder containing the email (default: INBOX)" }
    },
    "required": ["accountId", "uid"]
  }
}
```

**Handler success response:** `"Email ${uid} deleted from ${folder}."`
