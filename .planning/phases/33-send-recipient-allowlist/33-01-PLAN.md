# Phase 33 Plan 01: Send Recipient Allowlist

## Objective

Add an optional `allowedRecipients` field to email account config that restricts which addresses AI can send to. When set, all outgoing recipient addresses (to/cc/bcc) for send_email, create_draft, reply_email, forward_email are validated against the allowlist before any SMTP/IMAP I/O.

## Context

- @CONTEXT.md — feature design and rationale
- src/config.ts — emailAccountSchema (add allowedRecipients field)
- src/utils/validation.ts — add validateRecipients utility
- src/index.ts — dispatchTool() integration

## Tasks

### Task 1 — RED: Failing tests for validateRecipients (tdd="true")

Write failing tests in `src/utils/validation.test.ts` for the new `validateRecipients(recipients, allowlist, accountId)` function.

**Behavior:**
- Throws ValidationError when an address does not match the allowlist
- Error message includes the blocked address and account ID
- Accepts exact address match (case-insensitive)
- Accepts domain pattern match (@example.com matches any@example.com)
- Does not throw when all recipients are in the allowlist
- Does not throw for empty recipients string
- Does not throw for undefined recipients
- Handles comma-separated addresses (each validated individually)
- Handles angle-bracket format addresses

### Task 2 — GREEN: Implement validateRecipients

Add `validateRecipients(recipients: Array<string | undefined>, allowlist: string[], accountId: string): void` to `src/utils/validation.ts`.

Implementation:
- Reuse address parsing logic (split by comma, extract from angle brackets)
- For each address, check: exact match (case-insensitive) OR domain match (address ends with the pattern, e.g. `@example.com`)
- Throw `ValidationError` for first non-matching address with message: `"Recipient {addr} is not in the allowed recipients list for account {accountId}"`

### Task 3 — Integration: Wire allowedRecipients in config + dispatchTool

1. Add `allowedRecipients: z.array(z.string()).optional()` to `emailAccountSchema` in `src/config.ts`

2. In `src/index.ts` `dispatchTool()`, after the existing email format validation block, add allowlist validation:
   - Fetch the account from `getAccounts()` to get `allowedRecipients`
   - If `allowedRecipients` is set and non-empty, call `validateRecipients`:
     - `send_email`: validate to, cc, bcc
     - `create_draft`: validate to, cc, bcc
     - `forward_email`: validate to, cc, bcc
     - `reply_email`: validate cc, bcc only (to is auto-determined)
