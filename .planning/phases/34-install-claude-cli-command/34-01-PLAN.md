# Phase 34, Plan 01: Install Claude CLI Command

## Objective

Add `--install-claude` CLI flag to `mail-mcp` that writes/updates the MCP server config in Claude Desktop's `claude_desktop_config.json` for zero-friction one-command setup.

## Context

@CONTEXT.md

## Tasks

### Task 1: Write failing tests for installClaude (TDD RED)
type="auto" tdd="true"

**behavior:**
- `installClaude(configPath, binaryPath)` takes explicit paths (testable without touching real FS)
- Creates config dir if missing
- Creates config file with `mcpServers.mail` entry if file doesn't exist
- Merges into existing config, preserving other mcpServers
- Updates existing `mail` entry if already present
- Returns written config path
- Throws on malformed JSON in existing config

**test file:** `src/cli/install-claude.test.ts`

### Task 2: Implement installClaude function (TDD GREEN)
type="auto" tdd="true"

**implementation:**
- New file: `src/cli/install-claude.ts`
- Export `installClaude(configPath: string, binaryPath: string): Promise<string>`
- Use `node:fs/promises` for file operations
- Use `node:path` for path handling
- Merge logic: read existing → parse → set `config.mcpServers = config.mcpServers ?? {}` → set `config.mcpServers.mail = { command: binaryPath }` → write with 2-space indent

### Task 3: Add --install-claude flag to main() and wire helper
type="auto"

Wire the flag and binary path detection into `src/index.ts`:
- Add `'install-claude': { type: 'boolean', default: false }` to `parseArgs` options
- When flag is set: detect binary path → call `installClaude(desktopConfigPath, binaryPath)` → print success → exit 0
- Binary path detection: `which mail-mcp` (child_process execSync) → fall back to `process.argv[1]`
- Desktop config path: `path.join(os.homedir(), 'Library', 'Application Support', 'Claude', 'claude_desktop_config.json')`
- Also update help text to include `--install-claude` description

### Task 4: Run tests and verify

Run `npm test` and confirm all tests pass including new install-claude tests.
