# Phase 23 Plan 01: Header-Only Fetch

## Objective

Add `headerOnly` boolean parameter to `list_emails` so AI agents can scan large
mailboxes for metadata (subject, from, date, flags) without downloading message bodies.

## Context

@23-CONTEXT.md

## Tasks

### Task 1 — Write failing tests (TDD RED)
type: auto
tdd: true

Write `src/protocol/imap.headerOnly.test.ts` with unit tests for
`ImapClient.listMessages` `headerOnly` flag behaviour. Tests must fail before
implementation.

Behavior to test:
- `headerOnly=false` (default): fetch options include `bodyParts: ['TEXT']`; snippet is non-empty when bodyParts data present
- `headerOnly=true`: fetch options do NOT include `bodyParts`; snippet is empty string `''`
- Backward compat: calling with 4 args `(folder, count, offset)` works (no 5th arg)

### Task 2 — Implement headerOnly in ImapClient
type: auto
tdd: true

Modify `ImapClient.listMessages` signature:
```ts
async listMessages(
  folder = 'INBOX',
  count = 10,
  offset = 0,
  headerOnly = false
): Promise<MessageMetadata[]>
```

When `headerOnly=true`, omit `bodyParts: ['TEXT']` from fetch options and set `snippet = ''`.

### Task 3 — Thread MailService and MCP tool
type: auto

1. `MailService.listEmails` adds `headerOnly = false` as 4th param, forwards to
   `imapClient.listMessages`
2. `src/index.ts` `list_emails` tool schema: add optional `headerOnly` boolean field
3. `src/index.ts` dispatch: read `args.headerOnly`, pass to `service.listEmails`

### Task 4 — Run tests and verify
type: auto

Run `npm test` — all tests (including new ones) must pass.
