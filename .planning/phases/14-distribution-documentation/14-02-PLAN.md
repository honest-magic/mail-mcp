**Step 3: Create `Formula/mail-mcp.rb`**

Create the directory `Formula/` in the repo root and write `mail-mcp.rb` with this exact structure (per D-01 through D-05):

```ruby
class MailMcp < Formula
  desc "MCP server for AI-powered email access via IMAP and SMTP"
  homepage "https://github.com/honest-magic/mail-mcp"
  url "https://registry.npmjs.org/@honest-magic/mail-mcp/-/mail-mcp-1.1.0.tgz"
  sha256 "<COMPUTED_SHA256_FROM_STEP_1>"
  license "MIT"

  depends_on "node"

  def install
    system "npm", "install", *std_npm_args
    bin.install_symlink libexec.glob("bin/*")
  end

  def caveats
    <<~EOS
      To get started, add your first email account:
        mail-mcp accounts add
    EOS
  end

  test do
    assert_match "<MATCH_STRING_FROM_STEP_2>", shell_output("#{bin}/mail-mcp --help 2>&1", <EXIT_CODE_FROM_STEP_2>)
  end
end
```

Key requirements (from decisions and research):
- Class name: `MailMcp` (CamelCase of `mail-mcp`) -- per D-01, Pitfall 3
- URL: npm registry tarball -- per D-03
- `depends_on "node"` -- per D-04
- `*std_npm_args` with splat operator -- per Pitfall 5
- `libexec.glob("bin/*")` not `Dir["#{libexec}/bin/*"]` -- per State of the Art
- Caveats tells user to run `mail-mcp accounts add` -- per D-05
- No `require "language/node"` at top -- deprecated per State of the Art
- sha256 is 64 characters (SHA-256, not SHA-1) -- per Pitfall 1

**Note:** This file is created in THIS repo for convenience. The checkpoint task will guide the user to set up the homebrew-tap repo and copy/push this file there.
