# Phase 28 Plan 01: Audit Logging

## Objective

Add append-only JSONL audit logging of every MCP tool invocation, enabled via `--audit-log` CLI flag, with sensitive-field sanitization.

## Context

@.planning/phases/28-audit-logging/CONTEXT.md

## Tasks

### Task 1: Add AUDIT_LOG_PATH constant to config (type=auto)

Add `AUDIT_LOG_PATH` constant to `src/config.ts` pointing to `~/.config/mail-mcp/audit.log`.

**Done criteria:** `AUDIT_LOG_PATH` exported from `src/config.ts`, points to correct path.

### Task 2: TDD ‚Äî AuditLogger utility class (type=auto, tdd=true)

Create `src/utils/audit-logger.ts` and `src/utils/audit-logger.test.ts`.

**Behavior (RED tests):**
- `AuditLogger` constructor takes `logPath: string`
- `sanitizeArgs(args)` strips top-level keys matching /password|refreshtoken|clientsecret|token|secret|key|auth/i
- `log(entry)` appends a JSONL line to the file (async, creates file if missing)
- `log(entry)` is a no-op when logger is disabled (enabled=false)
- Each JSONL line is valid JSON parseable independently
- Log file receives one line per call with no trailing newline issues

**Implementation:**
- `AuditLogger` class with constructor `(logPath: string, enabled: boolean = true)`
- `sanitizeArgs(args: Record<string, unknown>): Record<string, unknown>` ‚Äî pure function (can be static)
- `async log(entry: AuditEntry): Promise<void>` ‚Äî appends JSON + '\n' using `fs.appendFile`
- Creates parent dir if missing (like `saveAccounts` does)

**Done criteria:** All tests pass via `npm test`.

### Task 3: Wire AuditLogger into MailMCPServer (type=auto)

1. Add `--audit-log` CLI flag to `parseArgs` in `main()` in `src/index.ts`
2. Pass `auditLogger` to `MailMCPServer` constructor (optional, default undefined)
3. In `setupToolHandlers()` `CallToolRequestSchema` handler: wrap `dispatchTool` call to record start time, call logger.log() after result with success/error/durationMs
4. Update help text to include `--audit-log`

**Done criteria:**
- `--audit-log` flag parsed and passed down
- `MailMCPServer` constructor accepts optional `AuditLogger`
- Every tool call path logs an entry (success and error cases)
- Existing tests still pass

### Task 4: Integration smoke test (type=auto)

Add one integration test case to `src/index.test.ts` (or a new file) that:
- Creates an `AuditLogger` with a temp file path
- Creates a `MailMCPServer` with the logger injected
- Calls `dispatchTool('list_accounts', false, {})`
- Asserts one line was written to the temp log file and it's valid JSON with correct shape

**Done criteria:** Test passes, confirms end-to-end logging works.
