# Phase 22 Plan 01: Mailbox Stats Tool

## Objective

Implement a `mailbox_stats` MCP tool that returns unread counts, total messages, and recent activity per folder using the IMAP STATUS command — without listing individual emails.

## Context

@22-CONTEXT.md

## Tasks

### Task 1 — Add failing tests (TDD RED)
**type:** auto
**tdd:** true

Write failing tests for:
- `ImapClient.getMailboxStatus(folders)` in `src/protocol/imap.test.ts`
- `MailService.getMailboxStats(folders?)` in `src/services/mail.test.ts`
- `mailbox_stats` dispatch in `src/index.test.ts`

Tests must fail (method does not exist yet).

### Task 2 — Implement `ImapClient.getMailboxStatus()`
**type:** auto

Add `MailboxStatus` interface and `getMailboxStatus(folders: string[])` method to `src/protocol/imap.ts`.

Uses `this.client.status(folder, { messages: true, unseen: true, recent: true })`.
Parallelized across folders with `Promise.all()`.
Per-folder errors caught and returned as `{ name, total: null, unread: null, recent: null, error: string }`.

### Task 3 — Implement `MailService.getMailboxStats()`
**type:** auto

Add `getMailboxStats(folders?: string[])` to `src/services/mail.ts`.
When `folders` is omitted/empty, calls `listFolders()` first and runs stats on all folders.
Delegates to `this.imapClient.getMailboxStatus(folders)`.

### Task 4 — Add `mailbox_stats` tool to `src/index.ts`
**type:** auto

Add tool definition (readOnlyHint: true) and dispatch handler.

Input schema:
```json
{
  "accountId": "string (required)",
  "folders": "array of strings (optional, default: all folders)"
}
```

Output: formatted text table + raw JSON.

### Task 5 — Run tests (TDD GREEN)
**type:** auto

Run `npm test` — all tests must pass.
