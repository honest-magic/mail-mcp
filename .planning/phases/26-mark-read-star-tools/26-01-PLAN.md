# Phase 26, Plan 01: Mark Read/Star Tools

## Objective

Add four semantic MCP tools — `mark_read`, `mark_unread`, `star`, `unstar` — that let AI agents
mark emails without knowing IMAP flag syntax. Each wraps `service.modifyLabels()`.

## Context

@.planning/phases/26-mark-read-star-tools/CONTEXT.md

## Tasks

### Task 1 (tdd=true): RED — failing tests for the four new tools

Write tests in `src/index.test.ts` that cover:
- Each of the 4 tools appears in `getTools(false)` output
- None of the 4 tools appear in `getTools(true)` (read-only mode)
- Each tool has `readOnlyHint: false, destructiveHint: true`
- Total tool count is now 24 (was 20); read-only count is now 16 (was 12)
- `dispatchTool` with `readOnly=true` returns `isError: true` for all 4 tools
- `dispatchTool` with `readOnly=false` calls `service.modifyLabels` with correct flags
- `mark_read` passes `addLabels=['\\Seen'], removeLabels=[]`
- `mark_unread` passes `addLabels=[], removeLabels=['\\Seen']`
- `star` passes `addLabels=['\\Flagged'], removeLabels=[]`
- `unstar` passes `addLabels=[], removeLabels=['\\Flagged']`

Tests MUST FAIL before implementation.

### Task 2 (tdd=true): GREEN — implement the four tools

In `src/index.ts`:

1. Add `mark_read`, `mark_unread`, `star`, `unstar` to the `WRITE_TOOLS` set
2. Add 4 tool definitions in `getTools()` after the `use_template` tool
3. Add 4 dispatch handlers in `dispatchTool()` after the `modify_labels` handler
4. Update the read-only instructions string to mention the new tool names

All tests MUST PASS after implementation.
