# Phase 24 Plan 01: Email Templates

## Objective

Add reusable email template support so AI agents can list configured templates, apply
`{{variable}}` substitution, and receive ready-to-use args for `send_email` / `create_draft`.

## Context

@24-CONTEXT.md

## Tasks

### Task 1 — Template utility: schema, cache, helpers (TDD)
type: auto
tdd: true

**Behavior (RED tests):**
- `emailTemplateSchema` validates correct template objects and rejects missing `id`/`name`/`body`
- `getTemplates()` returns empty array when file is missing
- `getTemplates()` parses and returns valid templates, skipping invalid ones with a console.error
- `applyVariables(body, vars)` replaces `{{name}}` with the value from vars
- `applyVariables` leaves unknown placeholders intact
- `applyVariables` replaces multiple occurrences of the same variable
- `applyVariables` handles no-placeholder body unchanged

**Implementation:**
- Create `src/utils/templates.ts` with:
  - `TEMPLATES_PATH` constant: `~/.config/mail-mcp/templates.json`
  - Zod `emailTemplateSchema` with fields: `id`, `name`, `subject?`, `body`, `isHtml?`, `accountId?`
  - In-memory cache with `fs.watch` invalidation (same pattern as `config.ts`)
  - `resetTemplatesCache()` exported for tests
  - `getTemplates()` async — reads disk, validates per-item, returns array
  - `applyVariables(template: string, vars: Record<string, string>): string` — replaces `{{key}}` globally

### Task 2 — MCP tools: list_templates and use_template (TDD)
type: auto
tdd: true

**Behavior (RED tests):**
- `list_templates` with no `accountId` returns all templates as JSON
- `list_templates` with `accountId` returns global templates + matching account templates
- `use_template` with valid `templateId` and variables returns merged args JSON
- `use_template` with unknown `templateId` returns `isError: true` response
- `use_template` applies variables to both subject and body
- `use_template` with optional `to`, `cc`, `bcc`, `accountId` includes them in output

**Implementation:**
- In `src/index.ts`, add to `getTools()`:
  - `list_templates` — `readOnlyHint: true`
  - `use_template` — `readOnlyHint: true`
- In `dispatchTool()`, add handlers for both tools
- Import `getTemplates` and `applyVariables` from `./utils/templates.js`
- `use_template` handler: find template by id, apply vars, return JSON args object
