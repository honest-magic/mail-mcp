# Phase 31 Plan 01: Sensitive Content Redaction

## Objective

Add an optional `--redact` CLI flag that, when enabled, masks credit card numbers, SSNs, passwords, and API keys in email content before it is returned to the AI.

## Context

@.planning/phases/31-sensitive-content-redaction/CONTEXT.md

## Tasks

### Task 1 — Write failing tests for redactSensitiveContent (TDD RED)

type="auto" tdd="true"

Create `src/utils/redact.test.ts` with tests for:
- Credit card number redaction (16 digits with spaces/dashes)
- SSN redaction (XXX-XX-XXXX)
- Password-in-context redaction (password: VALUE, pwd: VALUE, passwd: VALUE)
- API key redaction (sk-..., api_..., token_..., secret_...)
- No mutation when no patterns match
- Empty string input
- Multiple patterns in same text

Tests import `redactSensitiveContent` from `./redact.js`. Tests must FAIL before implementation.

### Task 2 — Implement redactSensitiveContent utility (TDD GREEN)

type="auto" tdd="true"

Create `src/utils/redact.ts` exporting:
```typescript
export function redactSensitiveContent(text: string): string
```

Patterns (all via regex, no external deps):
1. Credit card: `/\b(?:\d[ -]?){15}\d\b/g` → `[REDACTED CC]`
2. SSN: `/\b\d{3}-\d{2}-\d{4}\b/g` → `[REDACTED SSN]`
3. Password context: `/\b(password|pwd|passwd|pass)\s*[:=]\s*\S+/gi` → `password: [REDACTED]`
4. API key: `/\b(sk|api_|token_|secret_)[A-Za-z0-9_\-]{16,}\b/g` → `[REDACTED KEY]`

All tests must pass after implementation.

### Task 3 — Wire --redact flag into CLI and MailService

type="auto"

1. Add `redact` boolean parameter to `MailService` constructor
2. In `readEmail()`, when `redact` is true, call `redactSensitiveContent(content)` before returning
3. In `src/index.ts`:
   - Add `'redact': { type: 'boolean', default: false }` to parseArgs options
   - Add `--redact` to help text
   - Pass `redact` flag to `MailMCPServer` constructor
   - `MailMCPServer` passes `redact` to `MailService` instances

### Task 4 — Run full test suite

type="auto"

Run `npm test` and confirm all tests pass. Fix any failures.
