# Phase 32 Plan 01: Per-Tool-Type Rate Limits

## Objective

Extend the rate-limiting infrastructure to support two tiers — read (100/60s) and write (20/60s) — both per-account. Wire the correct tier in tool dispatch based on `WRITE_TOOLS`.

## Context

@.planning/phases/32-per-tool-type-rate-limits/CONTEXT.md

## Tasks

### Task 1 — TDD: Add failing tests for TieredRateLimiter
type="auto" tdd="true"

**Behavior:**

- `TieredRateLimiter` has `consumeRead(accountId)` and `consumeWrite(accountId)` methods
- `consumeRead` uses the read limiter (higher limit); `consumeWrite` uses the write limiter (lower limit)
- Both throw `QuotaError` when their respective limits are exceeded
- Read and write limits are independent — exhausting write does not affect read quota
- Both are per-account — exhausting account A's write does not affect account B's write
- Constructor accepts `readPoints`, `writePoints`, `duration` for testability
- Default constants: `DEFAULT_READ_RATE_LIMIT_POINTS = 100`, `DEFAULT_WRITE_RATE_LIMIT_POINTS = 20`

**Test file:** `src/utils/rate-limiter.test.ts` (append new `describe` block)

### Task 2 — Implement TieredRateLimiter
type="auto"

Add `TieredRateLimiter` class to `src/utils/rate-limiter.ts`:
- Export `DEFAULT_READ_RATE_LIMIT_POINTS = 100`
- Export `DEFAULT_WRITE_RATE_LIMIT_POINTS = 20`
- `TieredRateLimiterOptions`: `readPoints?`, `writePoints?`, `duration?`
- Internally owns two `AccountRateLimiter` instances (one for read, one for write)
- `consumeRead(accountId: string): Promise<void>`
- `consumeWrite(accountId: string): Promise<void>`

### Task 3 — Wire TieredRateLimiter in index.ts
type="auto"

Replace `AccountRateLimiter` with `TieredRateLimiter` in `src/index.ts`:
- Import `TieredRateLimiter` instead of (or in addition to) `AccountRateLimiter`
- Change `private readonly rateLimiter = new AccountRateLimiter()` to `new TieredRateLimiter()`
- In the rate-limit guard (two locations: `dispatchTool` ~line 753 and request handler ~line 1125):
  - Call `consumeWrite(accountId)` if `WRITE_TOOLS.has(name)` (or equivalent tool name variable)
  - Call `consumeRead(accountId)` otherwise
- Add index-level integration test in `src/index.test.ts` for rate-limit tier routing

## Verification / Success Criteria

- `npm test` passes with no regressions
- `TieredRateLimiter` tests all green
- Write tools are limited at 20 req/60s per account
- Read tools are limited at 100 req/60s per account
- Exhausting write quota does not block read tools
- Existing `AccountRateLimiter` class and tests unchanged
