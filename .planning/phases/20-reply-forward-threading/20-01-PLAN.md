# Phase 20 Plan 01: Reply & Forward Threading

## Objective

Add `reply_email` and `forward_email` MCP tools that set RFC 2822 threading headers
(`In-Reply-To`, `References`) so AI-composed replies appear in the original conversation
thread. Extend `SmtpClient.send()` to accept optional extra headers.

## Context

@.planning/phases/20-reply-forward-threading/CONTEXT.md

## Tasks

### Task 1: Extend SmtpClient.send() with extraHeaders (TDD)
type="auto" tdd="true"

**Behavior (RED):**
- `SmtpClient.send()` accepts a 7th optional parameter `extraHeaders?: Record<string, string>`
- When `extraHeaders` is provided, each key/value pair is passed to nodemailer via the
  `headers` field of `mailOptions`
- When `extraHeaders` is not provided (undefined), no `headers` field is added to `mailOptions`

**Implementation (GREEN):**
- Update `SmtpClient.send()` signature to add `extraHeaders?: Record<string, string>`
- Before calling `transporter.sendMail()`, if `extraHeaders` is truthy, set
  `mailOptions.headers = extraHeaders`

### Task 2: Add replyEmail() to MailService (TDD)
type="auto" tdd="true"

**Behavior (RED):**
- `MailService.replyEmail(uid, folder, body, isHtml, cc, bcc, includeSignature)` method
- Calls `fetchMessageBody(uid, folder)` to get the original parsed message
- Extracts `messageId` from `parsed.messageId`
- Extracts existing `references` from `parsed.headers.get('references')` (may be undefined)
- Builds `In-Reply-To` header from messageId
- Builds `References` header: if existing references, append messageId; else use messageId alone
- Extracts subject from `parsed.subject`, prepends "Re: " if not already present
- Extracts reply-to address from `parsed.from.value[0].address` (original sender)
- Calls `smtpClient.send()` with the reply-to address, Re: subject, body, isHtml, cc, bcc,
  and the `extraHeaders` `{ 'In-Reply-To': ..., 'References': ... }`
- Saves to Sent folder via `imapClient.appendMessage()`
- Applies signature via `applySignature()` before sending

**Implementation (GREEN):**
- Add `async replyEmail(uid, folder, body, isHtml, cc, bcc, includeSignature)` method
- Use `_cachedFetchBody()` internally (reuse cache)
- Handle missing messageId gracefully (send without threading headers if unavailable)

**Files:**
- `src/services/mail.test.ts` (new describe block)
- `src/services/mail.ts` (add replyEmail method)

### Task 3: Add forwardEmail() to MailService (TDD)
type="auto" tdd="true"

**Behavior (RED):**
- `MailService.forwardEmail(uid, folder, to, body, isHtml, cc, bcc, includeSignature)` method
- Fetches original parsed message
- Prepends "Fwd: " to subject if not already present
- Formats original message header block:
  ```
  \n\n--- Forwarded message ---\nFrom: <original from>\nDate: <original date>\nSubject: <original subject>\nTo: <original to>\n\n<original body text>
  ```
- Appends this forwarded block to the user's preamble body
- Applies signature
- Calls `smtpClient.send()` without threading headers (no In-Reply-To/References)
- Saves to Sent folder

**Implementation (GREEN):**
- Add `async forwardEmail(uid, folder, to, body, isHtml, cc, bcc, includeSignature)` method

**Files:**
- `src/services/mail.test.ts` (new describe block)
- `src/services/mail.ts` (add forwardEmail method)
