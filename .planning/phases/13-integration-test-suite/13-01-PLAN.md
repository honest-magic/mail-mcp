2. Add `test:integration` script to package.json scripts:
   ```json
   "test:integration": "vitest run --config vitest.integration.config.ts"
   ```

3. Create `vitest.integration.config.ts` at project root:
   ```typescript
   import { defineConfig } from 'vitest/config';

   export default defineConfig({
     test: {
       globals: true,
       environment: 'node',
       include: ['tests/integration/**/*.integration.test.ts'],
       globalSetup: ['tests/integration/setup.ts'],
     },
   });
   ```
   The `include` glob MUST NOT overlap with the unit config's `src/**/*.test.ts`.

4. Create `tests/integration/setup.ts` — the globalSetup file:
   - Import smtp-server using `createRequire` from `node:module` (CJS interop — smtp-server has no ESM exports field).
   - Create `SMTPServer` instance with `secure: false`, `authOptional: true`.
   - In `onData` callback: accumulate stream chunks into a string, push to module-scope `receivedMessages` array, call `callback()`.
   - In `setup(project: TestProject)`: listen on port 0 (ephemeral), await listen callback via Promise, read port from `server.address()`, call `project.provide('smtpPort', port)`.
   - In `teardown()`: close the server via `server.close()` wrapped in a Promise.
   - Add TypeScript module augmentation for vitest `ProvidedContext` to declare `smtpPort: number`.
   - Full pattern is in 13-RESEARCH.md Pattern 2 and Code Examples section.

5. Create `tests/integration/` directory (mkdir -p).

   a. **Test: delivers a message end-to-end without mocked transport**
      - Get port via `const port = inject('smtpPort')`
      - Create nodemailer transporter: `{ host: 'localhost', port, secure: false, ignoreTLS: true }`
      - Send email with `transporter.sendMail({ from: 'sender@test.local', to: 'recipient@test.local', subject: 'Integration test', text: 'Hello from integration test' })`
      - Assert `info.messageId` is defined (string, truthy)
      - Assert `info.accepted` contains `'recipient@test.local'`
      - Assert `info.rejected` is empty array

   b. **Test: sends HTML email with headers intact**
      - Same transporter setup
      - Send with `html: '<p>HTML body</p>'` and custom header `headers: { 'X-Test-Header': 'integration' }`
      - Assert `info.messageId` is defined
      - Assert `info.accepted` contains recipient

   c. **Test: handles multiple recipients**
      - Send to `to: 'a@test.local, b@test.local'`
      - Assert `info.accepted` has length 2

3. After creating the test, run both test commands to verify:
   - `npm run test:integration` — MUST pass (SMTP tests execute and succeed)
   - `npm test` — MUST pass and MUST NOT show any integration test output
