Create `tests/integration/imap.integration.test.ts`:

1. At the top of the file, BEFORE any describe block, add `vi.mock` to bypass macOS Keychain:
   ```typescript
   vi.mock('../../src/security/keychain.js', () => ({
     loadCredentials: vi.fn(() => Promise.resolve(process.env.TEST_IMAP_PASS)),
     saveCredentials: vi.fn(() => Promise.resolve()),
     removeCredentials: vi.fn(() => Promise.resolve()),
   }));
   ```
   This is necessary because `ImapClient.connect()` internally calls `loadCredentials(account.id)` which reads macOS Keychain. In CI, there is no Keychain — the password comes from `TEST_IMAP_PASS` env var instead.

2. Define credential check:
   ```typescript
   const hasImapCredentials = Boolean(process.env.TEST_IMAP_HOST);
   ```

3. Wrap ALL tests in `describe.skipIf(!hasImapCredentials)('IMAP operations (requires TEST_IMAP_HOST)', () => { ... })`.

4. Inside the describe block, set up the IMAP client in `beforeAll`:
   - Construct account config object from env vars: `TEST_IMAP_HOST`, `TEST_IMAP_PORT` (default 993), `TEST_IMAP_USER`, `TEST_IMAP_PASS`
   - Create `ImapClient` instance and connect
   - Store client reference for use in tests

5. In `afterAll`: disconnect the IMAP client (call `client.disconnect()` or equivalent cleanup).

6. Tests to include:

   a. **Test: lists mailboxes from server**
      - Call the method that lists folders/mailboxes
      - Assert result is an array with at least one entry
      - Assert INBOX exists in the list

   b. **Test: lists messages from INBOX**
      - Call list messages on INBOX
      - Assert result is an array (may be empty on a test account, but should not throw)
      - If messages exist, assert first message has a `uid` property that is a number

   c. **Test: search returns results for broad query**
      - Search with a broad criteria (e.g., all messages, or since a very old date)
      - Assert result is an array (validates the search path works against a real server)

7. Each test should have a reasonable timeout (e.g., 15000ms) since they involve real network I/O:
   ```typescript
   it('lists mailboxes from server', async () => { ... }, 15000);
   ```

8. Run `npm run test:integration` WITHOUT `TEST_IMAP_HOST` set — verify IMAP tests show as skipped and exit 0.
