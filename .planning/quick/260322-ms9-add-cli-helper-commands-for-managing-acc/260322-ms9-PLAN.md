<task type="auto">
  <name>Task 1: Create accounts CLI module and add saveAccounts to config</name>
  <files>src/cli/accounts.ts, src/config.ts</files>
  <action>
1. In `src/config.ts`, add a `saveAccounts(accounts: EmailAccount[])` function that:
   - Computes path as `path.join(os.homedir(), '.config', 'mail-mcp', 'accounts.json')`
   - Creates the directory with `fs.mkdirSync(dir, { recursive: true })` if it does not exist
   - Writes `JSON.stringify(accounts, null, 2)` to the file
   - Also export `ACCOUNTS_PATH` constant so CLI can reference it for display

2. Create `src/cli/accounts.ts` with a single exported async function `handleAccountsCommand(args: string[]): Promise<boolean>`.
   - Returns `true` if a CLI subcommand was handled (caller should `process.exit`), `false` if not a CLI command (caller starts MCP server).
   - Parse `args` (which is `process.argv.slice(2)`): if first arg is `accounts`, handle subcommand from second arg (`add`, `list`, `remove`). Otherwise return `false`.

   **`accounts list`:**
   - Call `getAccounts()` from config
   - If empty, print "No accounts configured." and the config file path
   - Otherwise, print a formatted table/list with columns: id, name, host, user
   - Use `console.log` for output (not `console.error`, since this is CLI output not MCP)

   **`accounts remove <id>`:**
   - Read third arg as the account ID. If missing, print usage and exit(1).
   - Call `getAccounts()`, find the account by id. If not found, print error and exit(1).
   - Remove account from array, call `saveAccounts()`.
   - Call `removeCredentials(id)` from keychain module (wrap in try/catch, warn if keychain entry not found but don't fail).
   - Print confirmation: "Account '<id>' removed."

   **`accounts add`:**
   - Use `node:readline/promises` (`import { createInterface } from 'node:readline/promises'`) to create an interactive prompt on `process.stdin`/`process.stdout`.
   - Prompt for each field in order:
     - `id` (string, required, validate not empty and not duplicate of existing account IDs)
     - `name` (string, required, default to id if empty)
     - `host` (string, required — IMAP host, e.g. imap.gmail.com)
     - `port` (number, required, default 993)
     - `user` (string, required — email address)
     - `authType` (choice: login or oauth2, default login)
     - `useTLS` (y/n, default y)
     - `smtpHost` (string, optional — press enter to skip, default to host with "imap" replaced by "smtp" if host contains "imap")
     - `smtpPort` (number, optional, default 587 if smtpHost provided)
     - `password` (string, required if authType=login — note: this is NOT stored in accounts.json)
   - For the password prompt, note in the prompt text that the password will be stored in the macOS Keychain, not in the config file.
   - After collecting all fields:
     - Build the `EmailAccount` object (without password)
     - Append to existing accounts via `getAccounts()` + push + `saveAccounts()`
     - Store password in keychain via `saveCredentials(id, password)` (only if authType is login and password was provided)
     - Print confirmation: "Account '<id>' added. Password stored in macOS Keychain."
   - Close the readline interface.

   **Unknown subcommand or `accounts` with no subcommand:**
   - Print usage: "Usage: mail-mcp accounts <add|list|remove>"
   - Exit with code 1.

With:
```typescript
import { handleAccountsCommand } from './cli/accounts.js';

async function main() {
  const args = process.argv.slice(2);

  // Check for CLI subcommands before starting MCP server
  const handled = await handleAccountsCommand(args);
  if (handled) {
    process.exit(0);
  }

  // No CLI subcommand — start MCP server
  const { values } = parseArgs({
    args,
    options: { 'read-only': { type: 'boolean', default: false } },
    strict: false,
  });

  const server = new MailMCPServer((values['read-only'] as boolean | undefined) ?? false);
  server.run().catch(console.error);
}

main().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

Note: The import for `handleAccountsCommand` should go at the top of the file with other imports. The `parseArgs` import from `node:util` already exists.
