  <action>
    Create the directory `.github/workflows/` then write `.github/workflows/ci.yml` with the following exact structure:

    ```yaml
    name: CI

    on:
      push:
        branches: [main]
      pull_request:
        branches: [main]

    jobs:
      ci:
        runs-on: ubuntu-latest

        steps:
          - uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Type check
            run: npx tsc --noEmit

          - name: Test
            run: npm test
    ```

    Notes:
    - Job name MUST be `ci` — the publish workflow references it via `needs: ci`
    - `node-version: '20'` matches engines field (>=18.0.0) and gives consistent CI environment
    - `npx tsc --noEmit` runs the type check without emitting files (no dist/ pollution in CI)
    - `npm test` runs `vitest run` per package.json scripts
    - `cache: 'npm'` speeds up subsequent runs
    - No macOS Keychain needed: test files mock the keychain
  </action>
