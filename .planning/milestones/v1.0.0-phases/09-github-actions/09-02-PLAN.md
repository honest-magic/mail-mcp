  <action>
    Write `.github/workflows/publish.yml` with the following exact structure:

    ```yaml
    name: Publish

    on:
      push:
        tags:
          - 'v*'

    jobs:
      ci:
        runs-on: ubuntu-latest

        steps:
          - uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Type check
            run: npx tsc --noEmit

          - name: Test
            run: npm test

      publish:
        needs: [ci]
        runs-on: ubuntu-latest

        steps:
          - uses: actions/checkout@v4

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              registry-url: 'https://registry.npmjs.org'
              cache: 'npm'

          - name: Install dependencies
            run: npm ci

          - name: Build
            run: npm run build

          - name: Publish to npm
            run: npm publish
            env:
              # NPM_TOKEN must be set as a secret in GitHub repo settings
              # (Settings -> Secrets and variables -> Actions -> New repository secret)
              NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
    ```

    Notes:
    - `on: push: tags: ['v*']` — triggers only on version tags (e.g. v1.0.0, v1.2.3)
    - `jobs.ci` is an INLINE job (not a reusable workflow reference) running the same steps as ci.yml — this satisfies GHA-03 without requiring `on: workflow_call:` in ci.yml
    - `needs: [ci]` on the publish job enforces GHA-03: publish cannot run if CI fails
    - `registry-url: 'https://registry.npmjs.org'` is required for actions/setup-node to configure .npmrc correctly
    - `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` — npm reads this env var when publishing; the comment documents the required secret
    - `npm run build` compiles TypeScript to dist/ before publish (publishConfig.access=public in package.json handles scoped public publish)
    - Do NOT add `--access public` to npm publish command — publishConfig in package.json already sets this
    - Do NOT use `uses: ./.github/workflows/ci.yml` — ci.yml has no `on: workflow_call:` trigger and GitHub Actions will reject it
  </action>
